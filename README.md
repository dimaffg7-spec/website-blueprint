# Website Blueprint - премиальный сайт под ключ с AI-агентом

Открытая методология: как одному человеку с AI-агентом для кодинга собрать премиальный сайт студийного уровня - с дизайн-системой, SEO, GEO (оптимизация под ИИ-поиск), формой заявок, юридической обвязкой и скоростью в зелёной зоне - за считанные дни, а не недели.

> **Где запускать.** Подойдёт любой AI-агент, умеющий читать файлы и выполнять команды: **Claude Code** (в терминале, десктоп-приложении или веб), **Cursor**, **VS Code** (с агентом), **Windsurf** и другие. «Чат» и «сессия» ниже - это любое такое окно, не обязательно терминал.

Это не «сделай красиво». Это промышленный **конвейер**: бриф → план → сборка → аудит → передача, с чеклистами и готовыми промптами на каждом шаге. Проверен на реальных коммерческих проектах (обезличено).

> **Стек-агностик.** Примеры даны на Next.js и Astro, но методология переносится на любой стек: чеклисты SEO/GEO/perf/конверсии и грабли - универсальны.

## 🚀 Быстрый старт - один промпт

Не хочешь разбираться в структуре - просто открой AI-агента (Claude Code, Cursor, VS Code…) **в пустой папке будущего сайта** и вставь этот промпт. Агент сам скачает методологию и поведёт тебя по шагам - расспросит про сайт, попросит материалы и соберёт всё в проект:

```
Склонируй методологию website-blueprint и веди меня по ней как персональный веб-продюсер.

1. Выполни: git clone https://github.com/dimaffg7-spec/website-blueprint
   Затем прочитай website-blueprint/README.md и все файлы в website-blueprint/docs/,
   website-blueprint/prompts/ и website-blueprint/.claude/skills/site-factory/SKILL.md.
2. Дальше действуй как скилл site-factory: веди меня по конвейеру шаг за шагом.
3. Начни с брифа дня 0 - БЕЗ КОДА. Расспроси меня:
   - про что сайт, для кого, какая цель (заявки / запись / продажа)?
   - какой хостинг (PHP / Node / только статика)?
   - какие материалы у меня есть - и ПОПРОСИ приложить их для контекста: тексты,
     фото, лого, реквизиты, реальные отзывы, ссылку на текущий сайт, референсы вкуса.
4. Собери мои ответы и материалы в папку business/, покажи план фаз и дождись,
   пока я его подтвержу.
5. Не выдумывай факты. Чего нет - помечай [уточнить] или спрашивай. Не начинай
   верстать, пока не собран бриф и я не одобрил план.
```

> Если твой агент не умеет `git clone` - скачай репозиторий кнопкой **Code → Download ZIP**, распакуй в папку проекта и в промпте замени пункт 1 на «прочитай файлы из папки website-blueprint/».

## Как пользоваться - три уровня

**1. Просто читать.** Начни с [`docs/00-pipeline-and-process.md`](docs/00-pipeline-and-process.md) - там весь конвейер. Дальше по слоям.

**2. Копировать промпты.** Идёшь по проекту, на каждом этапе берёшь готовый промпт из [`prompts/`](prompts/copy-paste-prompts.md) и вставляешь в свой AI-агент.

**3. Поставить скилл (Claude Code).** Скопируй `.claude/skills/site-factory/` в свой `~/.claude/skills/` - и агент будет сам триггериться на «сделай сайт», задавать бриф-вопросы и вести по конвейеру. Инструкция - в [`.claude/skills/site-factory/SKILL.md`](.claude/skills/site-factory/SKILL.md).

## Философия в одном экране

- **Хостинг раньше архитектуры.** Первый вопрос проекта - что за среда исполнения. Один незаданный вопрос = день переделки.
- **Диагноз по числам, не на глаз.** «Медленно» без цифр (вес страницы, Lighthouse, First Load) - не диагноз.
- **Три режима работы:** большой чат на сборку фаз / «одна правка = один чат» на полировку / отдельный свежий чат на аудит.
- **Премиум = контраст и тишина**, не спецэффекты. Тёмное - акцент-сцена, не фон всего сайта. «Премиум не торопится».
- **Портировать идею, не код.** Красивый компонент из чужого стека - это ТЗ, а не вставка. Порти на CSS, не тащи рантаймы.
- **Не выдумывать.** Цифры, отзывы, лица, юр-формулировки - только реальные. Один выдуманный факт рушит премиум-бренд.
- **Методология живая.** После каждого сайта дописывай выученные грабли обратно в методичку.
