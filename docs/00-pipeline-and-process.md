# 00 - Конвейер и процесс

Каркас всей методологии: как вести проект от пустой папки до сданного сайта.

## Конвейер «сайт под ключ» (режим заказчика)

Для сайта заказчику с нуля. Проверено: от пустой папки до готового к заливке за ~6 рабочих дней одного человека с AI-агентом.

### День 0 - Бриф (БЕЗ КОДА)
Собрать до старта, иначе простой в середине проекта. Подробный опросник - [`../templates/brief-template.md`](../templates/brief-template.md).
- **Хостинг заказчика** - что реально есть (shared/Node/Docker?), доступ к панели и DNS. ← **Урок №1: среда исполнения выясняется в день брифа, до выбора архитектуры.** Классический провал - спланировать форму под Node SSR, а у клиента shared-хостинг с PHP без Node → пивот посреди проекта.
- Контент: текущий сайт (тексты забрать **1:1**), фото, лого, регалии, отзывы (**реальные!**).
- Юр: реквизиты, оферта, политика ПДн, лицензии; собирает ли сайт персональные данные.
- Куда падают заявки: email / мессенджер / CRM.
- SEO-наследство: выгрузка проиндексированных URL (для карты 301-редиректов).
- Референсы: сайты, которые нравятся заказчику.

### День 1 - План и сборка MVP
- План фаз в `plans/` **с готовым промптом под каждую фазу** (любой новый чат подхватывает фазу без разогрева) - шаблон [`../templates/plan-9-phases.md`](../templates/plan-9-phases.md).
- Прогнать **«10 причин провала»** на план ДО кода (промпт в [`../prompts/copy-paste-prompts.md`](../prompts/copy-paste-prompts.md)) → блок рисков в план.
- Фазы: 0 каркас → 1 дизайн-система (токены + страница `/styleguide`-эталон) → 2-3 страницы → 4 форма (**под реальный хостинг!**) → 5 SEO-техничка → 6 аналитика → 7 юр-страницы.
- Дизайн-система - из письменного бренд-спека (см. [`02-design-system.md`](02-design-system.md)).

### День 2 - SEO/GEO-пакет
Отдельный аудит-чат: чего не хватает → реализация. Полный чеклист - [`03-seo-geo.md`](03-seo-geo.md). Файл стратегии: кластеры запросов, конкуренты, контент-план.

### День 3 - Дизайн-марафон с заказчиком
- Режим **«одна правка = один чат»**, после каждой правки - коммит с человеческим именем (откат любой итерации за 30 сек).
- Заказчику - живое превью; правки словами/скриншотами.
- Структуру блоков брать с работающего сайта заказчика, контент 1:1.

### День 4 - Форма и интеграции
[`05-forms-legal-deploy.md`](05-forms-legal-deploy.md). Тестовая заявка проходит насквозь.

### День 5 - Конверсия + анимации + performance
- Конверсионный аудит - [`06-conversion-audit.md`](06-conversion-audit.md).
- Пакет CSS-микроанимаций с `prefers-reduced-motion`.
- Картинки → WebP, шрифты self-host, **Lighthouse ≥95 Performance/SEO - гейт фазы, не «потом»**. См. [`04-performance.md`](04-performance.md).

### День 6 - Финал и передача
- Security review (ключи, ПДн-логи, доступ к конфигам) + «10 причин провала» на функционале.
- Чистка репо, финальный деплой, проверка формы на проде.
- Sitemap в вебмастеры (Яндекс - с регионом) + переобход.
- Передача: список хвостов заказчику + ретроспектива себе.

## Режим «живой сайт» (личный / долгая полировка)

Для сайта, который уже в проде, конвейер сменяется **волнами**: каждая волна = сессия с целью + ретроспектива (контент / деплой / performance / SEO / чистка). Приёмы те же, темп другой.

## Три режима работы с AI-агентом (каркас)

| Режим | Что это | Когда |
|---|---|---|
| **Большой чат** | Одна сессия тянет связку фаз по плану | День сборки |
| **Рой мелких чатов** | «Одна правка = один чат» + коммит после каждой | Дизайн-полировка с заказчиком |
| **Аудит-чат** | Свежий контекст без знания реализации | SEO / конверсия / performance аудит |

Почему три: большой держит контекст фаз; мелкие дают откат за 30 сек и не копят мусор; аудит **обязан** быть отдельным - тот, кто писал, своих дыр не видит.

## Универсальные процессные правила

- **Диагноз до правок - по числам** (вес страницы, First Load JS, Lighthouse, keyframe в видео).
- **План с промптом под каждую фазу** - новый чат подхватывает без разогрева.
- **«10 причин провала»** на план ДО кода - риски вшиваются в план.
- **Цикл после фазы:** ошибки → упущения → реальна ли проблема → свежий взгляд на diff → security.
- **Разделяй «делает AI / нужен человек»:** тексты/вёрстка - агент сразу; цены, реальные цифры, юр-формулировки - ждут владельца.
- **Коммит после каждой правки** с человеческим именем; добавлять в индекс поимённо, никогда не `git add -A` вслепую.
- **Ретро после сессии** → грабли фиксируются письменно и переживают чат.
