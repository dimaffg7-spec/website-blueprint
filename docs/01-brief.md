# 01 - Бриф (День 0)

Собирается ДО кода. Пропущенный вопрос здесь = потерянный день в середине проекта. Готовый опросник для заказчика - [`../templates/brief-template.md`](../templates/brief-template.md).

## Вопросы, без которых не начинать

1. **Хостинг и среда** (критично, урок №1): какой хостинг, что на нём исполняется - PHP? Node? только статика? Есть доступ к панели управления и DNS? От этого зависит вся архитектура формы и рендеринга.
2. **Текущий сайт**: есть? Тексты забираем 1:1, структуру рабочих блоков переносим (не выдумываем свою).
3. **Материалы**: фото, лого (в каком виде - вектор/растр, с прозрачностью?), регалии, отзывы - **реальные, с именами**.
4. **Юр-пакет**: реквизиты организации, оферта, политика ПДн, лицензии. Собирает ли сайт персональные данные через формы (тогда - режим согласия и уведомление регулятора).
5. **Заявки**: куда падают - email, мессенджер-бот, CRM? Если бот - создать заранее, получить токен и ID чата.
6. **SEO-наследство**: домен старый? Нужна выгрузка проиндексированных URL под карту 301-редиректов, чтобы не потерять позиции при переезде.
7. **Референсы вкуса**: какие сайты нравятся заказчику? Есть бренд-бук / фирменные материалы?
8. **Процесс решений**: кто утверждает дизайн и как быстро отвечает - влияет на темп дня 3.

## Каркас проекта после брифа

Разложить ответы в структуру, чтобы AI-агент имел контекст:
```
project/
├── AGENTS.md (или CLAUDE.md)   ← контекст проекта для агента: стек, деплой, правила
├── business/
│   ├── raw/          ← тексты с текущего сайта 1:1
│   ├── brand/        ← бренд-спек, референсы, палитра
│   ├── products/     ← услуги/продукты/тарифы
│   ├── audience/     ← ЦА, аватары, возражения
│   └── seo/          ← кластеры запросов, конкуренты, контент-план
└── plans/            ← план фаз проекта
```

## Почему бриф - без кода

Соблазн «сразу начать верстать» - главная ловушка. Пример из практики: форма спроектирована как серверный эндпоинт на Node, а у заказчика оказался shared-хостинг только с PHP. Результат - переписывание архитектуры формы посреди проекта, минус день. Один вопрос в день 0 это предотвращает.
