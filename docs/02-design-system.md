# 02 - Дизайн-система

Цель раздела - научить **собирать** дизайн-систему под любой бренд, а не скопировать один готовый стиль. Ниже метод + нейтральный пример токенов. Свой фирстиль подставляешь на его место.

## Фундамент: дизайн-система ДО вёрстки

- **Стиль - из письменного бренд-спека, не «из головы».** Все цвета (hex) извлекаются из боевых бренд-материалов, а не подбираются по памяти. Нет спека - сначала создать его (см. ниже «дизайн из референсов»), потом верстать.
- **Всё на токенах.** Палитра, шрифты, кривая анимаций, длительности - переменные (CSS custom properties / Tailwind `@theme`). Любая правка дизайна = правка токена, никаких цветов «инлайном». Это даёт дешёвые развороты: сменить всю тему (например тёмную на светлую по вкусу заказчика) - минуты, а не переверстка.
- **Эталонная страница `/styleguide` прямо на сайте** - показывает все токены, компоненты, состояния. Против неё сверяется каждая правка.

## Дизайн из референсов («укради дизайн» → спек)

Заказчик кинул скриншоты сайтов, которые нравятся. Метод: не копировать пиксели, а **извлечь систему** и зафиксировать письменно. Промпт - в [`../prompts/copy-paste-prompts.md`](../prompts/copy-paste-prompts.md). На выходе - файл `business/brand/style-spec.md`: палитра (hex по ролям), шрифты (заголовки/текст), кривая и длительности анимаций, правила («тёмное только в hero», «никаких резких градиентов»).

## Нейтральный пример токенов

Полный файл - [`../templates/design-tokens.example.css`](../templates/design-tokens.example.css). Это **пример структуры**, не готовый бренд - замени значения под свой.

```css
:root {
  /* Палитра по РОЛЯМ, не по названиям цветов */
  --color-bg:        #FAFAF8;  /* основной фон */
  --color-surface:   #FFFFFF;  /* карточки, приподнятые поверхности */
  --color-border:    #E7E4DE;  /* бордеры, разделители */
  --color-text:      #1A1A1A;  /* основной текст (почти чёрный, НЕ #000) */
  --color-text-muted:#6B6B6B;  /* приглушённый текст */
  --color-accent:    #3D5A80;  /* главный акцент - ЗАМЕНИ под бренд */
  --color-accent-2:  #5B7BA5;  /* блик/hover акцента */
  --color-scene:     #16181D;  /* тёплый почти-чёрный для тёмных сцен (НЕ #000) */

  /* Одна сигнатурная кривая на весь сайт */
  --ease-signature: cubic-bezier(0.22, 1, 0.36, 1);
  --dur-slow: 0.9s;   /* премиум не торопится: 0.6-1.1s */
}
```

## Принципы премиального вкуса (переносимы на любой бренд)

- **Тёмное = акцент-сцена (hero / подвал), не фон всего сайта.** Люкс даёт контраст, а не сплошная чернота. Тёмный - тёплый почти-чёрный (`#16181D`), не чистый `#000`.
- **Премиум не торопится:** длительности 0.6-1.1с, одна кривая на весь сайт.
- **Ремесло в деталях:** noise-текстура «бумаги», тонкие волосяные линии, деликатные блики - вместо кричащих градиентов и «AI-slop».
- **Текст почти-чёрный, не чистый чёрный.** Фон - тёплый off-white, не стерильный `#FFFFFF` для основного полотна.
- **Флюид-типографика** через `clamp()`; заголовки - плотный минус-трекинг; технические лейблы - uppercase с широким трекингом.
- **Шрифты self-host** (не CDN), `display: swap`, только нужные веса, нужные языковые сабсеты (для кириллицы - обязательно подключить).

## Дизайн-правила, выученные на практике

- **Не копировать компоненты из чужого стека - портировать идею на CSS.** 3D-tilt карточка из React+библиотека-анимаций → чистый CSS + маленький pointer-скрипт: эффект тот же, ноль JS-бандла. Референс = ТЗ, не код.
- **Тяжёлые рантаймы в hero (3D-движки, WebGL-виджеты) - нет.** Бьют Core Web Vitals. CSS решает то же дешевле.
- **Reveal-on-scroll только с гейтом по классу `.js`.** Без JS контент обязан быть видимым, иначе `opacity:0` = невидимый сайт для поисковика (SEO-смерть).
- **Каждой анимации - фолбэк `prefers-reduced-motion`; каждому hover-эффекту - проверка `pointer: coarse`** (на тач-устройстве статично).
- **`scale` на blur-слоях и `backdrop-blur` на движущихся элементах - под запрет** (дорогая перерисовка каждый кадр). Подробнее - [`04-performance.md`](04-performance.md).
- **Никаких цветов вне палитры.** Понадобился новый цвет - сначала в токены.

## Конверсионные дизайн-паттерны

- **Мобильный первый экран = заголовок + CTA** (портрет/картинка ниже). Иначе прямая потеря заявок.
- **Липкая CTA** внизу экрана на мобилке (прячется, когда открыта форма).
- Полоса цифр-триггеров с count-up - данные из конфига, не хардкод.
- Отзывы без фото → аватары-инициалы (**лица не выдумывать**).
- Cookie-плашка в угол, не поверх CTA.
