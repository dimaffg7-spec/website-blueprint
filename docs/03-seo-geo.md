# 03 - SEO + GEO

SEO - чтобы находили люди через поиск. GEO (Generative Engine Optimization) - чтобы тебя **цитировали ИИ-движки** (ChatGPT, Perplexity, Яндекс Нейро, Google AI Overviews). Второе - новый слой, который большинство сайтов игнорирует.

## Стратегия ДО вёрстки

- **Кластеры запросов**, принцип **1 страница = 1 кластер**.
- Честная оценка: высокочастотные запросы «в лоб» не взять - топ у доменов-монстров. Деньги в **B2B-лонгтейле** (слабее конкуренция, выше чек) + **инфо-запросах под блог** + **GEO-цитируемости**.
- **Специфика ИИ-поиска в рунете:** генеративные движки цитируют в основном топ обычной выдачи. Значит GEO не отменяет классическое SEO, а надстраивается: сначала попасть в топ-30, регион в вебмастере, хостинг с локальным IP.
- Одностраничник-визитка структурно не выйдет в топ по конкурентным запросам. Рычаг роста - контент (блог/статьи), перелинковка, посадочные под кластеры.

## Техничка (чеклист)

- [ ] Уникальные `title` ≤60 / `description` ≤160 **под кластер**, на каждую страницу; шаблон тайтла для дочерних.
- [ ] Canonical на каждой значимой странице, синхронный с политикой слешей (trailing slash) - без двойного каноникал на один контент. Человекочитаемые URL. Карта 301 со старых адресов (продублировать серверным конфигом на случай отключённого rewrite).
- [ ] OG/Twitter-теги + **og-image в PNG 1200×630** (SVG соцсети не отображают - частая ошибка).
- [ ] `sitemap.xml` + `robots.txt` средствами фреймворка; только реальные URL, **без хеш-якорей**; закрыть служебное (`/api/`, папки с логами).
- [ ] **Allow-лист ИИ-краулеров** в robots (см. ниже).
- [ ] Мета-robots: `index,follow,max-image-preview:large,max-snippet:-1,max-video-preview:-1` - разрешает ИИ полный сниппет.
- [ ] Verification-коды вебмастеров в metadata (Google + для рунета обязательно Яндекс).
- [ ] Семантика h1-h3, alt у картинок, страница 404.
- [ ] Отключить автолинковку email/телефонов, если ломает дизайн (`format-detection`).

### Allow-лист ИИ-краулеров (robots.txt)
Явно впустить (отдельными правилами поверх общего):
```
GPTBot, OAI-SearchBot, ChatGPT-User        # OpenAI
Google-Extended, Googlebot                  # Gemini + AI Overviews
ClaudeBot, anthropic-ai, Claude-Web         # Anthropic
PerplexityBot, Perplexity-User             # Perplexity
Applebot, Applebot-Extended                 # Apple Intelligence
Amazonbot, Bingbot, CCBot, YandexBot        # прочие крупные
```

## JSON-LD: граф сущностей, не отдельные блоки

Центральные построители в одном файле. Каждой сущности - свой `@id`, связывать через `{"@id": ...}`, **не дублировать поля**:

- **Person** (эксперт: должность, области знаний, награды, `sameAs` на все соцсети) ↔
- **Organization** (лицензия через `hasCredential`, реквизиты, `founder`) ↔
- **WebSite** ↔
- **Course / Product** (инструктор, что преподаёт, офферы с ценами) +
- **FAQPage** + **BreadcrumbList** + **WebPage с `dateModified`**.

Два правила:
- **FAQPage генерировать из того же массива, что рендерит видимый FAQ** на странице - один источник правды. Schema обязана соответствовать видимому контенту (иначе нарушение гайдлайнов).
- **Review / AggregateRating - осторожно:**
  - на `Person` / `Organization` - **нельзя** (self-serving отзывы поисковики банят ручными санкциями);
  - на `Course` / `Product` - **можно и полезно**, если отзывы **реальные и видимые на странице**. Рейтинг не выдумывать никогда.

## GEO-пакет (оптимизация под ИИ-движки)

- **`/llms.txt`** (выжимка для ИИ) + **`/llms-full.txt`** (полный факт-дамп: Q&A, цены, реквизиты, дата обновления). Факты 1:1 с JSON-LD, без противоречий.
- **Answer-first**: первые 2-5 предложений секции = готовый ответ; подзаголовки формулировать вопросом. Значительная доля ИИ-цитат берётся из вводных абзацев.
- **Факт-плотность**: не «отличная поддержка», а конкретика с числами, именами, датами. ИИ цитирует факты, не прилагательные.
- **Свежесть**: контент моложе 30 дней цитируется заметно чаще → видимая дата обновления + `dateModified` в schema.
- **E-E-A-T**: отдельная страница/домен эксперта как якорь Knowledge Graph (Person-schema), 301 со старых адресов автора.
- **Off-site**: UGC-статьи на профильных площадках (значимая доля цитат ИИ в рунете - оттуда), карточки в агрегаторах, бизнес-справочники, сущность в Wikidata.
- **Контент-план блога**: темы с приоритетами, формат статьи: H1 + FAQ + таблица + дата + Article-schema.
