# 04 - Performance

Гейт: **Lighthouse ≥95 Performance/SEO** - вшит в фазу, не «потом». Диагноз всегда по числам (вес страницы, First Load JS, число keyframe в видео), не «на глаз».

## Видео

- **Scroll-scrubbed hero** (эффект «видео проматывается скроллом», как на продуктовых страницах Apple): видео не проигрывается, а `currentTime` двигается по scroll-прогрессу внутри `requestAnimationFrame` с порогом ~1 кадра.
- **Keyframe на каждом кадре - обязателен для скраба.** Иначе браузер декодирует от далёкого keyframe и лагает на любом железе. Диагностика «лагает hero» начинается с `ffprobe` (проверить плотность keyframe). Переэнкод:
  ```bash
  ffmpeg -y -i in.mp4 -an -c:v libx264 \
    -g 1 -keyint_min 1 -x264-params "keyint=1:min-keyint=1:scenecut=0" \
    -crf 26 -pix_fmt yuv420p -movflags +faststart out.mp4
  ```
- **`poster` обязателен** к скраб-видео + `preload="metadata"` (качает только заголовок файла, не всё видео). Иначе чёрный экран до загрузки читается как лаг.

## Анимации

- **`scale` на blur-слоях - под запрет.** Анимация `scale` на элементе с `filter: blur(...)` заставляет браузер перерастеризовывать блюр каждый кадр. Только `translate` (x/y-дрейф) + `will-change: transform`.
- **`backdrop-blur` на движущихся элементах - под запрет** (full repaint каждый кадр: например 90fps→5fps на ленте из десятков карточек). Сплошной фон вместо блюра.
- **Гейтинг бесконечных фоновых анимаций.** Не держать десятки вечных анимаций активными вне экрана. Гейт: `IntersectionObserver` + `document.hidden` + `prefers-reduced-motion`.
- Анимации по умолчанию - **CSS + IntersectionObserver** (крошечный скрипт), а не тяжёлые JS-рантаймы.
- **WebGL/шейдеры** (если очень нужно): монтировать только у вьюпорта и **размонтировать** вне экрана (у браузера лимит на число WebGL-контекстов); при статической генерации - гард, чтобы не рендерить на сервере.

## Изображения

- Статический экспорт → сжимать заранее (Astro Image на сборке / разовый скрипт `cwebp`). Целевые ширины по назначению: контент-фото 800-1000px, крупные сканы для лайтбокса 1400px q80 (не меньше - иначе мыло на ретине), логотипы ~300px. Экономия часто в 20-50 раз (мегабайты → десятки КБ).
- Hero-картинка: `fetchpriority="high"`; всё ниже фолда: `loading="lazy" decoding="async"`.
- Логотипы с прозрачностью на тёмном фоне - только WebP (JPEG даёт ореол).
- **Карусели: постеры-фасады вместо живых iframe.** Десятки живых видео-плееров (iframe) в движущейся ленте убивают композитор. В карточке - статичный постер + кнопка play; живой iframe монтируется только в лайтбоксе по клику. Постер видео добывается через oembed API площадки (нужен User-Agent в запросе, иначе часто 403).
- **Marquee (бегущая лента) без рывков:** ширину посчитать один раз, дальше анимация чистым CSS на композитор-потоке; тройная пауза (вне вьюпорта / вкладка скрыта / hover); `pointer-events: none` пока едет.

## Шрифты и прочее

- Self-host, `display: swap`, нужные языковые сабсеты, только используемые веса.
- Вырезать `console.*` из прод-бандла; убрать `X-Powered-By`; `preconnect`/`dns-prefetch` к внешним CDN (видео-хостинг), которые подключаются ниже фолда.

## Аналитика (сквозная проверка)

Счётчик + цели на отправку формы. **Проверять цепочку целиком: согласие на cookie → счётчик → цель.** Классический баг: согласие пишется в `sessionStorage`, а аналитика читает `localStorage` → счётчик молчит у повторных визитов. Правило: **одно хранилище** для записи согласия и для чтения.

## Фазовый порядок оптимизации (по риску)

1. **Ноль изменения пикселей:** `loading=lazy`, `preload=metadata`, гейт анимаций, снятие `scale`, gzip. Катится сразу.
2. **Изображения:** конвертация в WebP + ресайз, удаление мёртвых ассетов. Обязательна визуальная проверка (ретина, лайтбокс).
3. **JS/бандл:** ленивая загрузка анимационной библиотеки, оптимизация импортов. По отдельной проверке (легко тихо сломать анимации переходов).
