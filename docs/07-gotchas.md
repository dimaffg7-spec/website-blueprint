# 07 - Грабли (проверять на каждом сайте)

Сводный чек-лист ошибок, собранных из ретроспектив реальных проектов. Прочитать ДО старта вёрстки, а не после.

1. **Reveal-on-scroll через `opacity:0` без гейта по `.js`** = невидимый сайт для поисковика (он видит скрытый контент). Гейт: контент видим без JS, анимация включается только при наличии класса `.js`.
2. **Скриншоты превью при прокрутке врут** (многие тулы рендерят с `scrollY=0`, а scroll-hero сбрасывает позицию). Верифицировать вёрстку через инспекцию DOM / computed styles, а не картинкой; для секции внизу - изолировать блок наверх.
3. **Сверять peer-dependencies** библиотек с версией фреймворка (частый источник «билд падает на ровном месте»).
4. **Cookie-согласие и аналитика должны читать одно хранилище** (иначе счётчик молчит у повторных визитов).
5. **OG-image только PNG 1200×630** (SVG соцсети не отображают).
6. **Юр-тексты и цифры - только подтверждённые.** Отзывы в schema - только реальные и видимые. Лица/аватары не выдумывать.
7. **Тяжёлые рантаймы (3D-движки, WebGL, лишний JS) в hero - нет.** CSS решает то же дешевле, Core Web Vitals целы.
8. **Серверные файлы (обработчик формы) на прод сами не попадают** при статической сборке - вести чек-лист «что на проде», заливать отдельно.
9. **Security-review инструменты могут требовать git-origin** - если падает, прогнать чек-лист вручную.
10. **Не `rm -rf` вслепую** - удаляемое сначала показать; временное выносить перемещением, а не удалением.
11. **Гонка сборки и dev-сервера за общий кэш** (при статическом экспорте) - остановить preview перед финальной сборкой; exit code проверять напрямую, не через пайп в `tail`.
12. **Добавлять в git поимённо**, не `git add -A` вслепую (параллельные правки не должны попасть в коммит).
13. **При удалении фичи - искать осиротевшие бинарники** (видео/постеры тянут мегабайты мёртвого веса в прод).
14. **Логотипы - разные правила по типу:** wordmark с прозрачностью обрезать по контенту; лого-плитку (залитый цветной квадрат) НЕ обрезать (растянется); сплошную подложку - убирать в прозрачность цвет-ключом, а не обрезкой.
15. **Резерв высоты в CSS-каруселях считать в `rem`, не `em`** (line-height обычно привязан к корню).
16. **Файлы с секретами (`.env`, конфиги токенов)** - читать только точечно (`grep "^VAR="`), не целиком; писать файловым инструментом, а не `echo` в shell.
17. **Серверный конфиг (`.htaccess` и подобное) может уронить сайт** на старом сервере - проверить совместимость до включения в деплой.
18. **ПДн-логи - над веб-корнем** + закрыть в robots (иначе персональные данные доступны по прямой ссылке).
