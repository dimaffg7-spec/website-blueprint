# 08 - Скиллы и инструменты

Что реально помогает при сборке сайта с AI-агентом. Главный вывод из практики: **ценность родилась не из «магических» скиллов, а из повторяемых процессов** (мультиагентный аудит, диагностика по числам, дизайн-система как артефакт). Скилл - лишь удобная упаковка процесса.

## Скилл этого блюпринта

**`site-factory`** ([`../.claude/skills/site-factory/SKILL.md`](../.claude/skills/site-factory/SKILL.md)) - оркестратор конвейера. Ставишь в Claude Code, триггерится на «сделай сайт для X», задаёт бриф-вопросы, создаёт план фаз, ведёт по методичке. Это переупаковка всего репозитория в исполняемый вид.

## Типы инструментов, которые пригодятся

Не привязка к конкретным продуктам, а **категории** - подбери аналог под свой стек и агент:

| Категория | Зачем | Примеры-ориентиры |
|---|---|---|
| **Дизайн-советник** | Что выбрать: стиль, палитра, пары шрифтов, UX-паттерны под тип продукта | базы дизайн-интеллекта, «design system» скиллы |
| **Дизайн-реализатор** | Как реализовать: готовые компоненты, темизация, accessibility | shadcn/ui-каталоги, Tailwind-генераторы |
| **Готовые компоненты как ТЗ** | Красивый эффект из чужого стека → портировать идею на свой | реестры React-компонентов, animation-библиотеки (как референс, НЕ как зависимость) |
| **Превью/верификация** | Проверять вёрстку после каждой правки (инспекция DOM надёжнее скриншота) | встроенные preview-инструменты агента, headless-браузер |
| **Сжатие изображений** | PNG/JPG → WebP на сборке | Astro Image, `cwebp`, sharp |
| **Security-review** | Проверка ключей/ПДн/доступов перед сдачей | встроенные security-скиллы, ручной чек-лист |
| **Конверсионный аудит** | Разбор лендинга по фреймворкам (см. [`06-conversion-audit.md`](06-conversion-audit.md)) | UX/UI-советники, свой промпт «5 линз» |

> 📦 **Конкретный набор инструментов с проверенными командами установки и скриптом-установщиком** - в [`09-toolkit-setup.md`](09-toolkit-setup.md) (`bash scripts/setup-tools.sh`).

## Методологические приёмы (важнее любого скилла)

- **План с готовым промптом под каждую фазу** - новый чат подхватывает фазу без разогрева.
- **«10 причин провала»** - прогнать на план ДО кода, риски вшить в план.
- **Референс-как-ТЗ** - кидаешь код/скриншот, агент портирует идею на твой стек без лишних рантаймов.
- **Три режима чатов** (сборка / полировка / аудит) - см. [`00-pipeline-and-process.md`](00-pipeline-and-process.md).
- **Ретро после сессии** → грабли письменно, переживают чат.

## Честная оговорка про «скиллы»

Соблазн собрать сайт «одним магическим скиллом» - иллюзия. На реальных проектах формально запускались единичные скиллы (обычно дизайн-советник для точечной задачи), а 95% ценности дали обычные циклы «прочитал → написал → проверил» плюс дисциплина процесса. Скилл `site-factory` полезен как **оркестратор дисциплины**, а не как замена думанию.
