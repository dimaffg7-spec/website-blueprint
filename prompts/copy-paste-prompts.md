# Готовые промпты (копипаст)

Вставляй в свой AI-агент на нужном этапе. Квадратные скобки `[...]` - заменить своими данными.

---

## 0. Автозапуск - скачать методологию и вести по ней

Самый быстрый старт. Открой агента в пустой папке будущего сайта и вставь:

```
Склонируй методологию website-blueprint и веди меня по ней как персональный веб-продюсер.

1. Выполни: git clone https://github.com/dimaffg7-spec/website-blueprint
   Затем прочитай website-blueprint/README.md и все файлы в website-blueprint/docs/,
   website-blueprint/prompts/ и website-blueprint/.claude/skills/site-factory/SKILL.md.
2. Дальше действуй как скилл site-factory: веди меня по конвейеру шаг за шагом.
3. Начни с брифа дня 0 - БЕЗ КОДА. Расспроси меня: про что сайт, для кого, какая цель;
   какой хостинг (PHP / Node / только статика); какие материалы есть - и ПОПРОСИ приложить
   их для контекста: тексты, фото, лого, реквизиты, реальные отзывы, ссылку на текущий
   сайт, референсы вкуса.
4. Собери ответы и материалы в папку business/, покажи план фаз, дождись подтверждения.
5. Не выдумывай факты. Чего нет - помечай [уточнить] или спрашивай. Не начинай верстать,
   пока не собран бриф и я не одобрил план.
```

---

## 1. Бриф (день 0)

```
Мы делаем сайт для [бренд/продукт]. Прежде чем писать код, задай мне вопросы дня 0
и не начинай верстать, пока не получишь ответы минимум про:
- хостинг и среду исполнения (PHP? Node? только статика? доступ к панели и DNS?);
- источник контента (есть текущий сайт? заберём тексты 1:1?);
- материалы (фото, лого, регалии, реальные отзывы);
- юр-пакет (реквизиты, оферта, политика ПДн, лицензии; собираем ли ПДн формой);
- куда падают заявки (email / мессенджер / CRM);
- SEO-наследство (старый домен? нужна карта 301?);
- референсы вкуса (какие сайты нравятся).
Разложи мои ответы в структуру business/ и предложи план фаз.
```

## 2. План фаз + «10 причин провала»

```
Составь план сборки сайта в plans/ - фазы от каркаса до деплоя, у КАЖДОЙ фазы:
цель, deliverable, критерий «сделано» и готовый промпт, которым эту фазу можно
запустить в отдельном чате без разогрева.
Затем прогони «10 причин провала»: 10 конкретных способов, которыми этот проект
может провалиться (лиды теряются, SEO при переезде падает, hero тяжёлый, форма не
под тот хостинг и т.д.), и вшей блок «Риски и предохранители» прямо в план.
```

## 3. Дизайн-система из референсов («укради дизайн» → спек)

```
Вот скриншоты/ссылки на сайты, которые нравятся заказчику: [референсы].
Не копируй пиксели - извлеки СИСТЕМУ и зафиксируй письменно в business/brand/style-spec.md:
- палитра (hex по ролям: фон, поверхности, текст, приглушённый текст, акцент, тёмная сцена);
- шрифты (заголовки / текст), начертания, трекинг;
- сигнатурная кривая и длительности анимаций;
- правила («тёмное только в hero», «никаких резких градиентов», «премиум не торопится»).
Потом собери из этого токены (CSS-переменные / Tailwind @theme) и страницу /styleguide.
Никаких цветов вне палитры.
```

## 4. Конверсионный аудит лендинга (5 линз + 2 проверки)

```
Разбери лендинг [URL] как конверсионный аудитор. Пройди 5 НЕЗАВИСИМЫХ линз,
каждую изолированно (не подглядывая в выводы других):
1. StoryBrand (SB7): 5 секунд / герой / проблема / план / призыв / шум.
2. Direct Response: таблица Оффер / Выгода / Доказательства / Возражения / CTA
   (по каждому: статус слабо-средне-сильно + что не так + что делать).
3. Ясность: понятно ли за 5 секунд, кто клиент и что предлагают.
4. Потери конверсии: пройди страницу по экранам сверху вниз [0]..[N] -
   где выпадает внимание / почему / что сделать.
5. Копирайтинг: заголовки, офферы, микрокопия.
Затем 2 проверки: (а) фактчек - нет ли выдуманных цифр/обещаний;
(б) соответствие бренду и голосу.
Синтез: диагноз одной фразой + топ-3 проблемы, подтверждённые ≥2 линзами.
Для каждого слабого заголовка дай 3 варианта. Где нет данных - пиши [нужны данные],
не выдумывай. В конце - приоритеты топ-7 (эффект × сложность) и трекер решений.
```

## 5. Performance-диагностика

```
Проведи performance-аудит сайта ПО ЧИСЛАМ, не на глаз:
- вес public/ и самых тяжёлых картинок; First Load JS по страницам;
- число keyframe в hero-видео (ffprobe), если есть scroll-скраб;
- число бесконечных фоновых анимаций и есть ли у них гейт по вьюпорту;
- есть ли gzip, lazy-loading ниже фолда, self-host шрифтов, prefers-reduced-motion.
Выдай план из 3 фаз по возрастанию риска (1: ноль изменения пикселей;
2: изображения; 3: JS-бандл) с прогнозом эффекта в числах. Гейт - Lighthouse ≥95.
```

## 6. Стартовый промпт «сделай сайт» (если не ставишь скилл)

```
В этом репозитории лежит website-blueprint - методология сборки премиального сайта.
Прочитай README.md и docs/ целиком. Затем веди наш проект строго по конвейеру,
начиная с брифа дня 0 (сначала вопросы, потом код). На каждом слое сверяйся
с чеклистами (дизайн / SEO-GEO / performance / форма-юр / грабли).
Стек проекта - [Next.js / Astro / чистый HTML]. Дизайн - [Premium-стиль по референсам /
свой бренд-спек]. Перед кодом задай уточняющие вопросы, пока не уверен на 95%.
Что не знаешь по контенту - спрашивай или помечай [уточнить], не выдумывай.
```
