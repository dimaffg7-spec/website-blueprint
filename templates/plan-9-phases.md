# Шаблон плана: 9 фаз

Один план = один сайт. Имя файла: `plans/YYYY-MM-DD-site-<имя>.md`. У каждой фазы - статус `[ ]` / `[WIP]` / `[x]`, цель, deliverable, критерий «сделано» и **готовый промпт запуска** (чтобы фазу можно было выполнить в отдельном свежем чате).

## Фаза 0 - Каркас `[ ]`
- Цель: пустой проект собирается и запускается.
- Deliverable: каркас на выбранном стеке, dev-сервер поднимается.
- Готово: открывается пустая страница без ошибок.
- Промпт запуска: _____

## Фаза 1 - Дизайн-система `[ ]`
- Цель: токены + страница `/styleguide`-эталон.
- Deliverable: палитра, шрифты, кривая анимаций в токенах; styleguide рендерит все компоненты.
- Готово: все цвета/шрифты берутся из токенов, styleguide полный.

## Фаза 2-3 - Страницы `[ ]`
- Цель: главная + ключевые страницы по реальному контенту.
- Готово: структура 1:1 с согласованной, контент не выдуман.

## Фаза 4 - Форма `[ ]`
- Цель: рабочий приём заявок ПОД РЕАЛЬНЫЙ ХОСТИНГ.
- Deliverable: обработчик + 3 канала (мессенджер/email/лог) + согласие ПДн.
- Готово: тестовая заявка проходит насквозь.

## Фаза 5 - SEO-техничка `[ ]`
- Цель: metadata, JSON-LD граф, sitemap/robots, canonical, og-image PNG.
- Готово: чеклист `docs/03-seo-geo.md` закрыт.

## Фаза 6 - Аналитика `[ ]`
- Цель: счётчик + цели + согласие на cookie.
- Готово: цепочка согласие → счётчик → цель проверена сквозь.

## Фаза 7 - Юр-страницы `[ ]`
- Цель: оферта, политика ПДн, согласия - дословно из документов.
- Готово: все ссылки в футере/форме ведут на реальные документы.

## Фаза 8 - GEO-пакет `[ ]`
- Цель: llms.txt / llms-full.txt, answer-first, факт-плотность, дата обновления.
- Готово: чеклист GEO из `docs/03-seo-geo.md` закрыт.

## Фаза 9 - Финал `[ ]`
- Цель: performance ≥95, security review, деплой, передача.
- Готово: pre-flight пройден, сайт на проде, хвосты переданы владельцу.
